6  The Basics with R

Biomedical Engineering
Applied Statistics

Before delving into the many statistical and graphical tools available for data analysis, we first need to become familiar with the basic logic of the R programming language. This is the purpose of this tutorial.


6.1 Objects

Everything stored in R is an object. An object may be a variable, a function, a dataset, or any other element created during an R session. Objects available in the R workspace can be inspected either by typing ls() in the Console or by checking the Environment tab.

The object shown in the Environment tab (Figure 6.1) was created after running the code described in the following section.


6.2 Variables

Variables can be created using either the = or <- operator. Although both operators can be used for assignment, there are functional differences between them. For example, <- has higher precedence than =, which may lead to unexpected results if both are mixed in the same line of code. The general recommendation in the R Style Documentation is to use <- for assignment:

Most usages of the equal sign for assignments do not cause runtime errors. Not all possible problems were eliminated, however. Thus the equal sign is not recommended, it is tolerated.

We will therefore use <- for assignments and = for specifying named arguments within functions. Examples will follow.

Let us create the variable mu_cog_x_speed, storing the mean of the five CoG speed values listed in Table 4.1. We will then check whether this variable is present in the R workspace and print its value:

mu_cog_x_speed <- (6.1 + 5.2 + 5.7 + 7.1 + 6.3) / 5 # creating a variable object in R
print(mu_cog_x_speed)                               # printing the value of the only object created
[1] 6.08
ls()                                                # listing objects in the workspace
[1] "mu_cog_x_speed"

The hash symbol # indicates that the text following it is a comment, not a piece of code to be executed.

The variable just created also appears in the Environment tab (Figure 6.1). To remove this variable from the R workspace, we can use the command rm() (e.g., rm("mu_cog_x_speed")), or click on the brush icon () in the toolbar of the Environment tab. These options can be used to remove one, several, or all variables from the workspace.


6.3 Functions

Functions work as they do in other programming languages: they are blocks of code that execute when called by the user. Built-in functions are not shown in the workspace, whereas custom functions created by the user are. We will return to custom functions later on in this tutorial.

Available functions and their documentation can be inspected under the Packages and Help tabs.

We will now compute the same mean value as before, this time using the mean() function available in the base package.

mu_cog_x_speed <- mean(c(6.1, 5.2, 5.7, 7.1, 6.3))
mu_cog_x_speed
[1] 6.08

There are two important points to notice here.

  1. The computed mean value, stored in the variable mu_cog_x_speed, can be inspected without explicitly using the print() command.
  2. The function mean() was applied to a vector containing the numbers 6.1, 5.2, 5.7, 7.1, and 6.3, which were concatenated with the c() function.


6.4 Data types

The following data types are commonly used in R.


  • Numeric
  • Integer
  • Character
  • Logical

This is the most common data type in R. Whenever decimal numbers are assigned to a variable, R stores them as numeric values by default. We can check this by creating the vector cog_x_speed, containing five plausible values for CoG speed in the lateral direction, using the c() function:

cog_x_speed <- c(6.1, 5.2, 5.7, 7.1, 6.3) # mm/s
class(cog_x_speed)
[1] "numeric"
str(cog_x_speed)
 num [1:5] 6.1 5.2 5.7 7.1 6.3

We used the class() and str() functions to inspect the type of the cog_x_speed variable. Notice that str() displays the data type, the dimension of the object, and the values stored in it.


Integer variables are similar to numeric variables, except that decimal values are not allowed. Suppose we want to store the age of the five subjects for whom cog_x_speed was computed. These values can be stored in a variable named age:

age <- as.integer(c(31, 25, 28, 47, 30)) # years
str(age)
 int [1:5] 31 25 28 47 30

By default, the vector created with the c() function stores numbers as numeric values. We explicitly converted them to integers using the function as.integer().


Character variables store text, also referred to as strings. Character values are created using quotation marks. In this example, we store the names of the five subjects for whom age and CoG speed values were obtained:

subjects <- c("Sarah", "Rob", "Ted", "Jessica", "John") # volunteers
str(subjects)
 chr [1:5] "Sarah" "Rob" "Ted" "Jessica" "John"


Logical variables can assume only two possible values: TRUE or FALSE. For example, we may ask whether each subject tested is younger than 30 years:

young_subjects <- age<30
young_subjects
[1] FALSE  TRUE  TRUE FALSE FALSE

According to this definition, two subjects are younger than 30. If we wish to identify these subjects, we can use the logical values stored in young_subjects as indices:

subjects[young_subjects]
[1] "Rob" "Ted"

In this case, R retrieves the second and third entries of the subjects variable because the second and third elements of young_subjects are TRUE. Alternative ways to access these two subjects are:

subjects[2:3]
[1] "Rob" "Ted"
subjects[c(2,3)]
[1] "Rob" "Ted"

The Parkinsonian column in Table 4.1 provides another example of a logical variable. We will recreate this variable below in the List section.


6.5 Data structures

There are different ways to organize data into R objects, depending on whether the variables of interest store numeric, text, or other types of data. Three useful functions help us understand the differences between data structures in R:

  • typeof(), which indicates the type of the stored data.
  • length(), which returns the number of elements stored.
  • attributes(), which returns additional information about the object.

The following sections briefly describe the most relevant data structures used in this course.


Vectors

Vectors are one-dimensional objects storing a collection of elements, all of the same data type. In other words, vectors are homogeneous data structures. A vector can store numeric data, such as the values stored in the cog_x_speed variable created earlier, or character data, such as subject names:

subjects <- c("Sarah","Rob","Ted","Jessica","John")
str(subjects)
 chr [1:5] "Sarah" "Rob" "Ted" "Jessica" "John"
typeof(subjects)
[1] "character"

The two variables subjects and cog_x_speed store collections of elements of the same type, character and numeric, respectively. In R, vectors may have one important attribute: names. Names can be defined when using the c() function:

two_subjects <- c(Sarah = 6.1, Rob = 5.2) # or c("Sarah" = 6.1, "Rob" = 5.2)
two_subjects
Sarah   Rob 
  6.1   5.2 
attributes(two_subjects)
$names
[1] "Sarah" "Rob"  
names(two_subjects)
[1] "Sarah" "Rob"  
str(two_subjects)
 Named num [1:2] 6.1 5.2
 - attr(*, "names")= chr [1:2] "Sarah" "Rob"

Here, we used the names() function to retrieve the names of the entries in the vector two_subjects. The same function can also be used to define or modify the names attribute.


Matrices

Like vectors, matrices store data of a single type, but they arrange these data in two dimensions. Instead of storing age and CoG speed in two separate variables, we can create a matrix storing both variables in two separate columns, using the matrix() function:

age_and_cog_x_speed <- matrix( 
  data = c(31, 25, 28, 47, 30, 6.1, 5.2, 5.7, 7.1, 6.3), 
  nrow = 5, ncol = 2,
  byrow = FALSE             # indicates whether entries should be arranged by rows
  ) 

typeof(age_and_cog_x_speed) # why is "double" the result of typeof()?
[1] "double"
age_and_cog_x_speed
     [,1] [,2]
[1,]   31  6.1
[2,]   25  5.2
[3,]   28  5.7
[4,]   47  7.1
[5,]   30  6.3

The result of typeof() is "double" because all elements in a matrix must have the same type. Since the vector used to create the matrix contains decimal values, R stores all entries as numeric double-precision values, including the ages.

Similarly, we may name either or both dimensions of the matrix just created, using the functions dimnames(), rownames(), or colnames():

colnames(age_and_cog_x_speed) <- c("age","cog_x_speed")
attributes(age_and_cog_x_speed)
$dim
[1] 5 2

$dimnames
$dimnames[[1]]
NULL

$dimnames[[2]]
[1] "age"         "cog_x_speed"
str(age_and_cog_x_speed)
 num [1:5, 1:2] 31 25 28 47 30 6.1 5.2 5.7 7.1 6.3
 - attr(*, "dimnames")=List of 2
  ..$ : NULL
  ..$ : chr [1:2] "age" "cog_x_speed"
NoteUtility of matrices

Matrices will rarely be used to analyze results from biomedical research directly. They can be very useful, however, when processing raw data. We will use matrices later to compute stabilometric descriptors from the sway data collected with smartphones.


List

The main distinction between a list and a vector or matrix is that a list can store data of different types. A list can also nest vectors or matrices of different sizes.

With the list() function, we can create a list grouping the data shown in Table 4.1. This can be done without naming the groups of elements:

# creating a list with subjects, CoG lateral speed, and Parkinsonian status 
subjects <- c("Sarah", "Rob", "Ted", "Jessica", "John") 
cog_x_speed <- c(6.1, 5.2, 5.7, 7.1, 6.3)
parkinsonian <- c(FALSE, FALSE, FALSE, TRUE, FALSE)
table_1 <- list(subjects, cog_x_speed, parkinsonian)
table_1
[[1]]
[1] "Sarah"   "Rob"     "Ted"     "Jessica" "John"   

[[2]]
[1] 6.1 5.2 5.7 7.1 6.3

[[3]]
[1] FALSE FALSE FALSE  TRUE FALSE
typeof(table_1) # type is "list", as different data types are stored in table_1
[1] "list"
length(table_1) # this is 3, as there are 3 elements in table_1
[1] 3
str(table_1) # this fully describes table_1
List of 3
 $ : chr [1:5] "Sarah" "Rob" "Ted" "Jessica" ...
 $ : num [1:5] 6.1 5.2 5.7 7.1 6.3
 $ : logi [1:5] FALSE FALSE FALSE TRUE FALSE

In this case, each element in the list table_1 stores five values. This does not have to be the case. Different elements in a list may store different numbers of values, either of the same type or of different types.

It is worth noticing that we could have assigned names to each element in the list:

table_1 <- list(subjects = subjects, cog_speed = cog_x_speed, is_parkinsonian = parkinsonian)
str(table_1)
List of 3
 $ subjects       : chr [1:5] "Sarah" "Rob" "Ted" "Jessica" ...
 $ cog_speed      : num [1:5] 6.1 5.2 5.7 7.1 6.3
 $ is_parkinsonian: logi [1:5] FALSE FALSE FALSE TRUE FALSE
attributes(table_1)
$names
[1] "subjects"        "cog_speed"       "is_parkinsonian"


Factor

A factor is a special data type in R. It is used to represent categorical data, typically stored as repeated text labels across different observations1. *In our example, observations are the five subjects, who may be categorized, for example, according to gender**:

gender <- factor(c("female", "male", "male", "female", "male"))
str(gender)
 Factor w/ 2 levels "female","male": 1 2 2 1 2

In this case, the gender factor() has five entries and two levels—that is, the set of values the factor may assume: "male" and "female". The levels of a factor can be accessed with the levels() function:

attributes(gender)
$levels
[1] "female" "male"  

$class
[1] "factor"
levels(gender)
[1] "female" "male"

We may add this factor to the table_1 list just created. Notice that we need to wrap it in another list before appending it; otherwise, we would not obtain the desired result. What would happen in that case?

table_1 <- append(table_1, list(gender = gender), after = 1) # insert at specific location
table_1
$subjects
[1] "Sarah"   "Rob"     "Ted"     "Jessica" "John"   

$gender
[1] female male   male   female male  
Levels: female male

$cog_speed
[1] 6.1 5.2 5.7 7.1 6.3

$is_parkinsonian
[1] FALSE FALSE FALSE  TRUE FALSE
str(table_1)
List of 4
 $ subjects       : chr [1:5] "Sarah" "Rob" "Ted" "Jessica" ...
 $ gender         : Factor w/ 2 levels "female","male": 1 2 2 1 2
 $ cog_speed      : num [1:5] 6.1 5.2 5.7 7.1 6.3
 $ is_parkinsonian: logi [1:5] FALSE FALSE FALSE TRUE FALSE

Notice that we used the append() function instead of c(). To remain consistent with the columns in Table Table 4.1, we used after = 1 to place gender as the second element of table_1.

Also notice how clearly the str() function summarizes the content of the table_1 object, indicating the number of elements stored, the name of each element (which could be NULL), the type of data, the number of values, and the corresponding values within each entry.


6.6 Data Frames

A data frame is another R structure for organizing data. Instead of describing it alongside the other structures covered in the previous section, it is presented in a stand-alone section. This is because data frames are the most common data organization structure used in R for statistical analysis.

Data frames are tabular structures—that is, they organize data in two dimensions. Rows correspond to observations, while columns store the variables2 associated with each observation.

An observation is generally defined as a piece of information collected experimentally. In biomedical research, observations often correspond to persons. Taking the example of Table 4.1, which is redrawn here (cf. Table 6.1) to explicitly indicate the observations, experimental data were obtained from five subjects. The variables of interest are Gender, Lateral CoG Speed, and whether the subject is affected by Parkinson’s disease. Subject and Age store personal data3: the name of each individual is rarely useful in statistical analysis, whereas age may be relevant when evidence suggests that it may influence the variables of interest (e.g. does CoG speed depend on age?).


Creating Data Frames

Table 6.1, from columns Subject to Parkinsonian, can be created using the data.frame() command, based on the R variables we created in previous sections:

df <- data.frame(subjects, gender, age, cog_x_speed, parkinsonian)
df
  subjects gender age cog_x_speed parkinsonian
1    Sarah female  31         6.1        FALSE
2      Rob   male  25         5.2        FALSE
3      Ted   male  28         5.7        FALSE
4  Jessica female  47         7.1         TRUE
5     John   male  30         6.3        FALSE

The tabular structure of the data frame object df is clear. Importantly, rows were identified with integers, from 1 to 5, and columns were named after the R variable objects passed to the data.frame() function. The name of each variable (column) may be defined explicitly when calling data.frame(). Using short but meaningful names is advisable.

df <- data.frame(sub = subjects, sex = gender, age = age, cog = cog_x_speed, disease = parkinsonian)
str(df)
'data.frame':   5 obs. of  5 variables:
 $ sub    : chr  "Sarah" "Rob" "Ted" "Jessica" ...
 $ sex    : Factor w/ 2 levels "female","male": 1 2 2 1 2
 $ age    : int  31 25 28 47 30
 $ cog    : num  6.1 5.2 5.7 7.1 6.3
 $ disease: logi  FALSE FALSE FALSE TRUE FALSE

The str() function provides a useful summary of the whole data frame structure. It tells us:

  • how many observations we have (\(n=5\)),
  • the variables associated with each observation,
  • the name of each variable ("sub", "sex", "age", "cog", "disease"),
  • the type of data stored in each column (chr, Factor, int, num, logi),
  • the data itself.


Accessing Entries in Data Frames

Elements stored for a single variable in the data frame df can be retrieved in different ways, using single brackets [], double brackets [[]], or the accessor $.

With single brackets [], we access one specific variable of the data frame df—that is, one of its five columns:

df[1] # this is a data frame with one variable ("sub") and five ("chr") observations
      sub
1   Sarah
2     Rob
3     Ted
4 Jessica
5    John
df[3] # this is a data frame with one variable ("age") and five ("int") observations
  age
1  31
2  25
3  28
4  47
5  30
str(df[3])
'data.frame':   5 obs. of  1 variable:
 $ age: int  31 25 28 47 30

To retrieve the entries of a single column as a vector, use double brackets [[]] or the accessor $:

df[[3]] # vector of integers 
[1] 31 25 28 47 30
str(df[[3]])
 int [1:5] 31 25 28 47 30
df$age # equivalent to df[[3]]
[1] 31 25 28 47 30


Retrieving Information About Data Frames

Some useful R functions can be used to inspect the structure of a data frame:

nrow(df) # number of rows (observations)
[1] 5
ncol(df) # number of columns (variables)
[1] 5
dim(df[1:3]) # number of rows and columns
[1] 5 3
length(df) # number of rows or columns? What do you think?
[1] 5

Often, a data frame contains many observations. In such cases, printing the full content of the data frame (df in our case) may be confusing, and we may lose sight of the column names. The head() function can help:

head(df, 3) # checking the first three observations in df
    sub    sex age cog disease
1 Sarah female  31 6.1   FALSE
2   Rob   male  25 5.2   FALSE
3   Ted   male  28 5.7   FALSE

The summary() function is remarkably useful for obtaining a brief overview of descriptive statistics for the whole df object. We will discuss descriptive statistics in detail later (cf Chapter 10). In our example, we have five variables, with the first serving only for illustrative purpose. For each of the following four variables we can quickly inspect:

  • Gender: the levels of this factor and the number of observations in each level,
  • Age and Lateral CoG Speed: minimum and maximum values, first, second, and third quartiles, and the mean value,
  • Parkinsonian: how many subjects are affected by Parkinson’s disease and how many are not.

This is achieved with summary():

summary(df)

6.7 Importing data

We will import data frequently, often with the purpose of addressing questions of interest, such as:

  • Does this new training program improve gait function?
  • Is muscle spasticity relieved by a serious-game rehabilitation program?
  • How much does this new method for counting motor units outperform existing approaches?

Addressing these questions requires the use of statistical tools to analyze variables. These variables may be obtained directly or after processing raw data for a group of subjects, according to a carefully planned experimental protocol. We may therefore have to import two types of files: files with raw data4 and files with variables5.

NoteWhich data will we work with?

For now, we will consider the former case, working with raw data. Upon completion of this tutorial, instead of working with simulated data or data collected by others, we will work with postural data collected by ourselves and from ourselves.


Getting Folder and File storing CoG data

In this course, PhyPhox is used to collect CoG data (cf Chapter 3).

The first step is to identify the folder where the raw data of interest are stored. For now, we are working with an example of CoG data, which you can download and find inside this compressed here.

Unzip the downloaded file and place its contents inside a folder named phyphox_data, in your Working Directory (cf. Files section in the previous Chapter).

my_wd <- getwd() # this command retrieves my working directory my_wd
my_wd
list.dirs(my_wd, recursive = FALSE, full.names = FALSE) # with list.dirs() we list the files inside our working directory

The function list.dirs() listed all directories in my_wd, retrieved with the getwd() function. We see that there is only one phyphox_data directory inside my_wd. We may further check the content of phyphox_data:

list.files(file.path(my_wd, "phyphox_data"))
[1] "standing_eyes_closed"     "standing_eyes_closed.xls"
[3] "standing_eyes_open"       "standing_eyes_open.xls"

With file.path(), we created a path to the phyphox_data directory by concatenating "phyphox_data" to my_wd. With the list.files() function, we listed the full content of the phyphox_data directory, including files and directories.

There are two xls files and two directories inside phyphox_data. All were obtained after exporting data from PhyPhox (see Section 3.4.2.0.5). Each folder is associated with an xls file—that is, the file "standing_eyes_open.xls" contains the same information as the file "postural_sways.csv" inside the folder "standing_eyes_open":

list.files(file.path(my_wd, "phyphox_data", "standing_eyes_open"))
[1] "meta"               "postural_sways.csv"


Loading CoG data into R workspace

To import CoG data into R, we may proceed interactively by right-clicking on the csv (or xls) file under the Files tab. Two options appear: View File and Import Dataset... (Figure Figure 6.2). By clicking on View File, a preview of the data is shown in the Source Pane if it is a csv file. If it is an xls file, R opens the file in Excel.

By clicking on Import Dataset..., a dialog may appear asking the user to install a package for importing the data, if the package is not already installed. If the required package is available, a window opens with useful and self-explanatory information on the data import procedure (Figure @ref(fig:figure-2-3-3)).

For now, we will import the csv data using the read.csv() function, already available in the utils package. The content of the csv file is loaded into the R workspace as a data frame. In the next Chapter, we will plot and process these raw CoG data, extracting the variables of interest for further analysis.
