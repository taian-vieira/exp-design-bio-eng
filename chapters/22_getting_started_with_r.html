5  Getting Started with R

Biomedical Engineering
Applied Statistics

All activities in this course will be conducted using the R programming language. R is free, open source, and was originally developed for statistical computing and data visualization.

As stated on the R Project website, R is

an environment within which statistical techniques are implemented.

The same source further emphasizes that:

One of R’s strengths is the ease with which well-designed publication-quality plots can be produced, including mathematical symbols and formulae where needed.

R has become one of the standard languages for statistical analysis in biomedical research, as reflected by its widespread use in scientific reports and peer-reviewed articles.

The preference for R is likely motivated by:

  • the ease with which data can be manipulated in R
  • the numerous libraries specifically designed and freely available for advanced statistical analysis
  • the graphical facilities available for visualizing and exploring data
  • the possibility of using standard programming tools, such as conditionals, loops, user-defined functions, and data import/export
  • many additional tools available within the R ecosystem, including R Markdown and Quarto for writing reproducible documents, books1, and articles,as well as shiny for developing interactive web applications

A brief tutorial on the R programming language will be presented in the following chapters, focusing only on the aspects that are most relevant for this course. The specific use of R for designing experiments and analyzing experimental data will be explored through ad hoc exercises.


5.1 Installation (R and RStudio)

Start by following the two steps indicated on the Posit website:

  1. install R
  2. install RStudio

RStudio is an open-source integrated development environment (IDE) for using R, although it can also be used with other languages, such as Python. For students who do not wish to install the desktop version, RStudio can also be used through Posit Cloud2.


5.2 RStudio IDE

After R and RStudio have been correctly installed, RStudio can be launched. The RStudio IDE interface should then appear, with four main panes, as shown in Figure 5.1.

The following sections provide a brief explanation of the features of interest in each pane.


Pane 1

This pane may not appear when RStudio is launched for the first time. It displays readable files, such as txt, md, Rmd, qmd, css, json, tex, yml files, among others. Of particular interest are R files, which are plain-text files containing executable R code. We will return to R files in Chapter 7.

Briefly, the Source Pane is where users can view and edit code-related files.


Pane 2

This pane is used to execute R code interactively. In the example shown in Figure 5.1, a numeric variable named x has been created and the value 1 has been assigned to it. Notice that, unlike in many other programming languages, the operator <- is commonly used for assignment in R instead of =. Chapter 6 introduces some basic R features, including variable definition, data types, data structures, and data import.


Pane 3

There are two main tabs of interest in this Pane:

Environment

The Environment tab displays all currently saved R objects. After running the command x <- 1 in Pane 2 (Console), the variable x is saved in the R Global Environment. From the moment the variable x has been defined, it can be inspected in the Environment tab (cf. Pane 3 in Figure 5.1).

Other potentially useful features of the Environment tab include:

  • interactive import () of datasets from text files, Excel files, or common statistical software formats (e.g., SPSS).
  • display of memory usage.
  • the brush icon (), which removes all objects from the workspace (Global Environment).
  • loading and saving () data into and from the R workspace.

History

In this tab, all commands executed during the current R session can be inspected. It is also possible to save and load history data, as well as to erase the entire history or only part of it.


Pane 4

There are four main tabs of interest in this Pane:

Files

This tab is self-explanatory. It allows users to navigate through directories, create new folders or files, delete files, and rename them. One important concept introduced here is the Working Directory. This is the directory R uses when executing code, and it is not necessarily the same directory currently shown in the Files tab.

To set a Working Directory, click on the () icon on the top-right edge of the Files tab. Then, navigate to the directory you wish to set as your Working Directory, select it, click on the () icon, and then click on Set As Working Directory (Figure 5.2).

The importance of the Working Directory will be discussed in Chapter 7.

Plots

All static figures created during an R session are stored in this tab. Users can navigate through the figures created, export them, delete all or some of them, and zoom in or out. This tab provides a useful graphical interface for exploring datasets.

Packages

Packages are R libraries containing functions, datasets, documentation, and, in some cases, compiled code. Packages extend the capabilities of R for specific tasks, such as data manipulation, visualization, or inferential statistics.

The Packages tab lists all installed packages. A package can be used only after it has been installed. To install a new package, most of which are available from the Comprehensive R Archive Network, use the command install.packages().

Help

In this tab, users can find documentation for packages and functions.

You can navigate through previously opened help pages using the left and right arrows (). By clicking on the home icon (), the general help page appears, providing many useful links.

To open the documentation for a specific function, you may either use the help(function_name) command or the syntax ?function_name. Consider, for example, the function mentioned in the previous section, install.packages(). To check its documentation, you may type either:

?install.packages
help(install.packages)
